# Embroidery & Threads Client Launch Checklist

## Client confirmations still required

- Replace the temporary phone number everywhere after the client supplies the correct number.
- Confirm desired inquiry response time, ordering boundaries, deposits, turnaround time, cancellations, and pickup expectations.
- Confirm permission to publish each customer photo, review screenshot, quote, and social handle currently shown.
- Have the client review and approve the website privacy policy.
- Claim or update the Google Business Profile after the client confirms the final phone number, service area, hours, categories, and photos.

## Client handoff still required

- Invite the client to the editor with the account she will use long term.
- Confirm the client can sign in, replace one image, publish it, and view the result without assistance.
- Transfer or share appropriate Netlify, Cloudflare, GitHub, Google Search Console, and Google Business Profile access.
- Enable two-factor authentication on all owner and administrator accounts.
- Record recovery codes in the client's password manager.
- Submit one contact-form test with the client present and confirm delivery to `embroideryandthreads@gmail.com`.

## Completed production setup

- Client editor is enabled at `https://embroideryandthreads.com/admin/`.
- Netlify Identity registration is invite-only and Git Gateway is enabled.
- Google Search Console domain ownership is verified and the sitemap is accepted.
- Bing Webmaster Tools has imported the verified site and accepted the sitemap.
- Cloudflare Web Analytics is enabled without individual visitor tracking.
- Netlify form notifications route new inquiries to `embroideryandthreads@gmail.com`.
- Custom-domain HTTPS, the `www` redirect, public routes, admin route, and true 404 responses are monitored every six hours.
- Desktop and mobile browser checks cover navigation, forms, image lightbox behavior, responsive overflow, and serious accessibility violations.
- Build checks cover editable image requirements, metadata, canonical URLs, robots directives, required public files, analytics, form detection, and asset budgets.
- Dependabot and a clean dependency audit cover known package vulnerabilities.
- A monthly GitHub issue prompts a real form-delivery test.
